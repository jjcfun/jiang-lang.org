Jiang 0.2.2 provides a prebuilt macOS arm64 compiler package. This is the recommended way to try the language today.

## Requirements

- macOS on arm64.
- LLVM 21.1.x from Homebrew `llvm@21`.
- A host C compiler available as `cc` for final executable linking.

The release package does not bundle `libLLVM.dylib`; it uses the local LLVM 21 installation at runtime.

## Install a Release

Download `jiang-0.2.2-macos-arm64.zip` from the [0.2.2 GitHub release](https://github.com/jjcfun/jiang/releases/tag/0.2.2), then unpack it.

Install LLVM first:

```bash
bash ./script/install_llvm_macos.sh
```

If Homebrew is missing, the script prints the Homebrew install command and exits without changing the system.

Install Jiang:

```bash
./install.sh
```

By default this installs:

```text
~/.jiang/versions/0.2.2
~/.jiang/bin/jiangc
```

Add Jiang to your shell path if needed:

```bash
export PATH="$HOME/.jiang/bin:$PATH"
```

## Verify

Check the compiler and LLVM runtime:

```bash
jiangc --version
```

The first line should be:

```text
jiang 0.2.2
```

## Build From Source

Source builds are mainly for compiler development. Clone the repository:

```bash
git clone https://github.com/jjcfun/jiang.git
cd jiang
```

Install LLVM:

```bash
bash ./script/install_llvm_macos.sh
```

Build the next compiler with the installed bootstrap compiler:

```bash
VERIFY=full bash ./script/build_next.sh
```

The 0.3 development branch expects a Jiang 0.2.2 compiler on `PATH` as `jiangc`, or an explicit
`BOOTSTRAP_BIN=/path/to/jiangc`.
