Jiang 用不同的类型后缀区分拥有资源的指针、普通引用、裸指针、多元素裸指针和切片。这里先介绍
这些类型的写法和基本操作；资源移动和生命周期规则见[所有权、借用与生命周期](/zh/language/ownership/)。

## 指针类别

- `T^`：`Box<T>` 的短写，表示拥有所有权的指针，通常来自 `new T(...)`。
- `T&`：`Reference<T>` 的短写，表示不拥有所有权的引用，不参与释放。
- `T*`：`RawPointer<T>` 的短写，表示单对象裸指针，用于外部函数边界、二进制接口和低层能力。
- `T[*]`：`ManyPointer<T>` 的短写，表示多元素裸指针，可通过下标访问连续地址中的元素。
- `T[*:S]`：`SentinelPointer<T, S>` 的短写，表示带哨兵值的多元素裸指针。
- `T[]`：`Slice<T>` 的短写，表示切片目标类型。
- `T[]&`：`Reference<Slice<T>>` 的短写，表示带运行时长度的借用切片引用。
- `T[:S]&`：`Reference<SentinelSlice<T, S>>` 的短写，表示带哨兵值的借用切片引用。

只有 `T^` / `Box<T>` 表达语言级所有权。`T&`、`T[]&`、`T[*]` 和 `T*` 都不拥有目标对象。
所有权转移、自动析构和生命周期约束在独立章节中展开。

## 创建所有者和引用

```c
Int value = 123;

Int^ owner = new Int(123);
Int[3]^ array_owner = new [1, 2, 3];

Int& ref = value$.ref();
Int* raw = value$.ptr();
```

`^`、`&` 和 `*` 都是类型后缀，紧跟在被包裹的类型后面：

```c
Int^ owner;
Int& ref;
Int* raw;
```

## 与可选和可变组合

类型后缀仍按从左到右、从里到外阅读：

```c
Int!^? maybe_owner; // 可为空的拥有所有权指针，指向可变 Int
Int?& maybe_ref;    // 指向可空 Int 的引用
Int^! owner_slot;   // 拥有所有权的指针变量本身可重新赋值
```

同一层级同时出现可选和可变时，规范顺序是 `?` 再 `!`，例如 `Int?!`。

## 解引用和成员访问

`T^` 不会在赋值、初始化或算术这类普通值上下文中自动解引用。需要读取指向的值时，使用 `$.get()`：

```c
Int^ make_value();

Int^ owner = make_value();
_ inferred = make_value();        // inferred: Int^
Int copied = make_value()$.get();
Int sum = make_value()$.get() + 1;
```

成员访问是例外：`owner.member` 会自动穿过 `T^` 访问被拥有对象的成员。

```c
struct Box {
    Int value;
}

Box^ box = new Box(value: 42);
Int n = box.value;
```

`T&` 和 `T*` 不默认自动解引用，需要使用 `$.get()` 显式读取：

```c
Int value = 41;
Int& ref = value$.ref();
Int copied = ref$.get();

Int* raw = value$.ptr();
Int raw_value = raw$.get();
```

写入单对象裸指针指向的目标时，被指向值的顶层必须可变：

```c
Int! value = 0;
Int!* raw = value$.ptr();
raw$.set(42);
```

## 隐式操作层

`$` 会阻止接收者自动解引用，并进入隐式操作层。`$.get()`、`$.ref()`、`$.ptr()`、`$.move()`、
`$.free()`、`Type$.size()` 这类操作都在独立章节中说明：见[隐式操作层](/zh/language/implicit-layer/)。

## 多元素裸指针

`T[*]` 是裸指针的一种，表示一段可下标访问的连续地址。它没有长度，不参与默认自动解引用：

```c
UInt8[*] bytes;

UInt8 first = bytes[0];
bytes[1] = 42;
```

和 C 不同，`T*` 是单对象裸指针，不用来表达“指针加偏移后访问任意位置”。需要按位置访问
连续内存时，使用 `T[*]`；需要带长度的连续内存借用视图时，使用 `T[]&` 切片。
