Jiang 的数组长度是类型的一部分，必须在编译期确定，不支持运行时改变长度。

## 不可变数组

```c
Int[3] a = [1, 2, 3];

// 长度不匹配会导致编译错误
// Int[5] a = [1, 2, 3];  // 错误

// 在堆上创建 owning pointer
Int[3]^ x = new [1, 2, 3];

// owning pointer 自动解引用，直接访问 length
print("array %d", x.length);
```

## 类型推断

使用 `_` 或 `Int[_]` 让编译器推断数组长度：

```c
// 以下三种方式等价
_ a = [1, 2, 3];
Int[_] b = [1, 2, 3];
Int[3] c = [1, 2, 3];
```

## 数组可变性

数组长度固定，但元素可变性和绑定可变性可以独立控制：

```c
// 元素可变
Int![_] b = [1, 2, 3];
b[1] = 4;  // b: [1, 4, 3]

// 绑定可变，元素不可变
Int[_]! c = [1, 2, 3];
c = [4, 5, 6];

// 元素和绑定都可变
Int![_]! d = [1, 2, 3];
d[1] = 4;
d = [4, 5, 6];

// owning pointer 绑定可变
Int[_]^! e = new [1, 2, 3];
e = new [4, 5, 6];
```

## 多维数组

Jiang 没有真正的多维数组——多维数组就是嵌套数组。以 `Int[2][3]` 为例：从里往外看，最外层有 3 个元素，每个都是 `Int[2]` 类型。

```c
Int[2][3] foo = [[1, 2], [3, 4], [5, 6]];

foo[0];      // [1, 2]
foo[0][1];   // 2
```

类型的阅读顺序是 **从左往右，从里到外**：`Int -> Int[2] -> Int[2][3]`，类型范围逐渐扩大。简单来说：**越靠右边，范围越大**。

指针类型也遵循同样规则：

```c
Int?[2][3] b = [[1, null], [3, 4], [5, 6]];
Int?[2]?[3] c = [[1, null], null, [5, 6]];

Int[3]^ b = new [1, 2, 3];
Int?[3]^ c = new [1, null, 3];
```

## 切片 (Slice)

切片 `T[]` 是一个带有 `length` 属性的胖指针，长度在运行时确定。可以把它看作 `{ ptr, len }` 的轻量视图，不表达所有权语义。

```c
Int[_] x = [1, 2, 3];
Int[] y = x[..];

print("y.length = %d", y.length);  // 输出：y.length = 3
```

切片下标访问返回元素引用：

```c
_ item_ref = y[1]$.ref();  // item_ref: Int&
