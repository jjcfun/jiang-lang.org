泛型让同一份代码适用于不同类型。泛型参数写在声明名之后。

## 泛型函数

```c
T id<T>(T value) {
    return value;
}

Int a = id<Int>(42);
_ b = id(42);
```

当调用上下文足够明确时，类型参数可以由实参和目标类型推导。

## 泛型类型

```c
struct Pair<A, B> {
    A first;
    B second;
}

Pair<Int, UInt8[]> pair = Pair<Int, UInt8[]> {
    first: 1,
    second: "one",
}
```

泛型实参也是类型，因此可以继续携带可选、可变、数组、指针等后缀：

```c
Pair<Int?, UInt8[]> maybe_pair;
Pair<Int!, Int[3]^> mutable_and_owner;
```

## `@where`

`@where(...)` 写在泛型声明之前，用来约束泛型参数：

```c
@where(T: Numeric)
T add<T>(T left, T right) {
    return left + right;
}
```

多个约束用逗号分隔：

```c
@where(T: Hashable, U: Equatable)
Pair<T, U> make_pair<T, U>(T left, U right);
```

同一个类型参数需要同时满足多个 trait 时，使用 `&`：

```c
@where(T: Hashable & Equatable)
UInt hash_key<T>(T value);
```

## 类型相等约束

`Name == Type` 表示泛型参数必须等于某个具体类型：

```c
@where(T == UInt8)
Int use_byte<T>(T value);
```

相等约束使用 `==`，不是赋值符号 `=`。

投影关联类型相等约束写成 `T.[Trait].Assoc == Type`：

```c
@where(T.[Iterable].Item == UInt8)
Int count_bytes<T>(T value);
```

## 关联类型绑定

`trait` 约束可以带关联类型绑定：

```c
@where(T: Iterable<Item = UInt8>)
Int count<T>(T value);
```

这比把关联类型约束拆开更容易读，也让 `trait` 的要求集中在一个位置。

## 顶层可变泛型

默认情况下，泛型参数不自动接受带顶层 `!` 的类型实参。需要要求顶层可变时，使用内建 `Mutable` 约束：

```c
@where(T: Mutable)
struct Slot<T> {
    T value;
}

Slot<Int!> slot = Slot<Int!> { value: 1 }
```

`Mutable` 只检查顶层 `!`。例如 `Int!` 满足，`Int` 不满足；`Int![3]` 的顶层是数组层，是否满足取决于数组层本身是否带 `!`。

## 与 Errorable 组合

泛型函数可以返回 `T@E`。`@E` 位于返回类型边界，错误类型本身不在 `@` 后追加 `?` 或 `!`：

```c
@where(T: Numeric)
T@ParseErr parse_number<T>(UInt8[] text);
```

如果成功值需要可选或可变，把后缀写在 `@` 左侧：

```c
T?@ParseErr parse_optional<T>(UInt8[] text);
```

## 使用边界

- 泛型参数只接受类型，不支持非类型参数。
- 后置 `T id<T> @where(...)` 不支持，约束写在声明之前。
- 关联类型投影约束使用 `T.[Trait].Name == Type` 这种写法。
