Jiang 类型按“从左到右、从里到外”的方式阅读。每个类型后缀都会包裹它左侧已经形成的类型。

## 基本类型

常见基本类型包括：

```c
Bool flag = true;
Int count = -123;
UInt size = 123;
UInt8 byte = 255;
Int16 small = -45;
Float value = 12.3;
Double precise = 132.54;
Char ch = 'a';
```

`Int` 和 `UInt` 是指针宽度整数。需要固定 ABI、文件格式或网络协议布局时，使用 `Int8`、`Int16`、`Int32`、`Int64` 或 `UInt8`、`UInt16`、`UInt32`、`UInt64`。

`enum` 的 discriminant 默认使用 `Int32` 存储；只有显式声明其他整数表示时才会改变。

`()` 是单元类型。它是零大小值，也用于表示函数没有有意义的返回数据。

字符串字面量按 UTF-8 字节序列处理。默认借用视图类型是 `UInt8[:0]&`，因此编译器可以在
C 风格边界暴露以 0 结尾的数据，同时把切片长度和哨兵字节分开处理。

```c
UInt8[_] bytes = "jiang";
UInt8[:0]& view = "jiang";
```

字符串字面量在期望类型是普通 `UInt8[]&` 的位置仍可以直接赋值。对于非字面量的
哨兵切片，需要用 `slice()` 或强制转换到 `UInt8[]&` 显式转换：

```c
UInt8[:0]& c_view = "jiang";
UInt8[]& plain = c_view.slice();
UInt8[]& forced = c_view$.as(UInt8[]&);
```

## 类型后缀

类型后缀越靠右，包裹范围越大：

```c
Int[2][3] matrix;
Int?[2][3] nullable_items;
Int?[2]?[3] nullable_rows;
```

可以把 `Int[2][3]` 读成：

1. `Int`
2. `Int[2]`：两个 `Int` 的数组
3. `Int[2][3]`：三个 `Int[2]` 的数组

常见后缀如下：

| 表面语法 | 规范类型名 | 含义 |
| --- | --- |
| `T!` | `T` 加可变标记 | 当前层级可变 |
| `T?` | `Option<T>` | 可选值 |
| `T^` | `Box<T>` | 拥有所有权的指针 |
| `T&` | `Reference<T>` | 不拥有所有权的引用 |
| `T*` | `RawPointer<T>` | 单对象裸指针 |
| `T[*]` | `ManyPointer<T>` | 多元素裸指针 |
| `T[*:S]` | `SentinelPointer<T, S>` | 带哨兵值的多元素裸指针 |
| `T[N]` | `Array<T, N>` | 定长数组 |
| `T[N:S]` | `SentinelArray<T, N, S>` | 带哨兵值的定长数组 |
| `T[]` | `Slice<T>` | 切片目标类型 |
| `T[]&` | `Reference<Slice<T>>` | 借用切片引用 |
| `T[:S]` | `SentinelSlice<T, S>` | 带哨兵值的切片目标类型 |
| `T[:S]&` | `Reference<SentinelSlice<T, S>>` | 借用哨兵切片引用 |
| `T@E` | `Result<T, E>` | 返回类型位置的可错返回 |

这些表面语法都是更短的写法。类型检查时会按规范类型名理解它们；即使用户自己定义了同名类型，也不会改变
`T?`、`T^`、`T&`、`T[]&` 或 `T@E` 这些语法的含义。日常代码优先使用短语法，讲解类型关系或写
`extend` / `@where` 时可以使用规范类型名。

同一层级里 `?` 必须写在 `!` 前面，例如 `Int?!`。`Int!?`、`Int??`、`Int!!` 都应诊断为错误。

## 类型推导

`_` 是右侧表达式类型的占位符。局部绑定会先计算初始化表达式的自然类型，忽略来源绑定携带的
最外层可变性，然后应用 `_` 后面写出的绑定后缀：

```c
_ answer = 42;
_ name = "Jiang";
_ values = [1, 2, 3];
```

`_` 后面的后缀属于新绑定，不属于右侧表达式：

| 绑定形式 | 含义 |
| --- | --- |
| `_ name` | 使用 RHS 类型创建不可变值绑定 |
| `_! name` | 使用 RHS 类型创建可变值绑定 |
| `_& name` | 借用 RHS place，创建不可变 `T&` 绑定 |
| `_&! name` | 借用 RHS place，创建可变引用槽位，槽位中存放 `T&` |

如果新绑定本身需要重新赋值，使用 `_!`：

```c
_! count = 0;
count = count + 1;
```

类型推导会保留表达式的自然所有权和引用形态，不会为了推导局部变量自动解引用 `T^` 或 `T&`：

```c
Int^ make_value();

_ owner = make_value(); // owner: Int^
Int copied = make_value()$.get();
```

右侧绑定最外层的可变性会被刻意忽略。`_` 描述被绑定值的类型；`_` 后面的 `!` 描述新绑定是否可变：

```c
Int! source = 1;
_ copied = source;   // 不可变绑定，值类型是 Int
_! mutable = source; // 可变绑定，值类型是 Int
```

使用 `_&` 可以从 place expression 推导出借用引用绑定：

```c
Int value = 1;
_& borrowed = value; // borrowed: Int&
Int copied = borrowed$.get();
```

`_&` 是 binding mode。它会主动借用右侧 place，并推导被引用的类型。右侧必须是可以被借用的
place；如果表达式本身已经是引用，应该使用 `_`，而不是 `_&`：

```c
Int& existing = value$.ref();
_ same_ref = existing;  // same_ref: Int&
// _& nested = existing; // 错误：`_&` 不会对已有引用再次借用。
```

如果引用绑定本身需要重新赋值，使用 `_&!`。这里的 `!` 只作用于绑定最外层，不要求被借用的源值可写：

```c
Int first = 1;
Int second = 2;
_&! current = first;
current = second$.ref();
```

同一组 `_` 绑定形式也用于解构。承载值模式中需要借用时，使用 `ref T name`：

```c
(_& left, _! right) = pair;

if maybe is .some(ref Int item) {
    use(item);
}
```

推导只决定最外层绑定。数组元素、元组元素、联合携带值或结构体字段如果需要可变性，要在对应类型位置显式写出 `!`。

## 分层可变性

`!` 标记当前层级可变。在绑定声明里，最外层的 `!` 标记绑定槽位本身，不属于槽位中存放的值类型。
进入复合类型内部后，`!` 才属于对应的嵌套值类型：

```c
Bool! flag = true; // 可变绑定槽位，槽位中存放 Bool
flag = false;

Int[3]! values = [1, 2, 3]; // 可变绑定槽位，槽位中存放 Int[3]
values = [4, 5, 6];

Int![3]! mutable_items = [1, 2, 3]; // 可变绑定槽位，槽位中存放 Int![3]
mutable_items[0] = 10;
mutable_items = [4, 5, 6];
```

例如，`Int! value` 声明的是“可变绑定，值类型为 `Int`”；`Int![]` 中的 `!` 位于数组元素层级，
因此数组元素类型可变。`Int[]! value` 则是“可变绑定，值类型为 `Int[]`”。

对于聚合值，成员是否可变由成员自己的类型决定。外层绑定可变，不代表所有字段或元素自动可变。

```c
struct User {
    Int id;
    Int! age;
}

User user = User(id: 1, age: 18)
user.age = 19; // 允许
// user.id = 2; // 编译错误
```

赋值、初始化、传参和 `return` 都是把值写入目标位置的场景。这里以目标类型为准，在相同结构位置忽略可变标记，但不会忽略可选层、指针种类、数组长度或字段结构差异。

## 数组和切片

数组长度是类型的一部分：

```c
Int[3] values = [1, 2, 3];
Int[_] inferred = [1, 2, 3];
```

嵌套数组通过继续添加后缀表达：

```c
Int[2][3] matrix = [[1, 2], [3, 4], [5, 6]];
```

`T[]` 是 `Slice<T>` 的短写，表示切片目标类型。需要运行时长度的借用切片引用时，使用 `T[]&`，
也就是 `Reference<Slice<T>>`：

```c
Int[_] values = [1, 2, 3];
Int[]& view = values[..];
```

`T[:S]` 是 `SentinelSlice<T, S>` 的短写。`T[:S]&` 这类带哨兵值的切片视图额外携带类型层面的哨兵值保证。非字面量哨兵切片
不会在赋值给 `T[]&` 时隐式丢掉这个保证；需要普通切片视图时，写成 `value.slice()`
或 `value$.as(T[]&)`。

## 指针和引用

指针和引用分成几类：

- `T^`：`Box<T>` 的短写，表示拥有所有权的指针，通常来自 `new T(...)`。
- `T&`：`Reference<T>` 的短写，表示不拥有所有权的引用。
- `T*`：`RawPointer<T>` 的短写，用于外部函数边界、二进制接口和低层能力。
- `T[*]`：`ManyPointer<T>` 的短写，可通过下标访问连续内存元素。

```c
Int^ owner = new Int(42);
Int value = owner$.get();
owner$.free();
```

`T^`、`T&` 和 `T*` 在普通值上下文中都不会自动解引用，需要用 `$.get()` 显式读取。成员访问是例外，`owner.member` 会自动穿过 `T^`：

```c
Int value = 41;
Int& ref = value$.ref();
Int copied = ref$.get();

struct Box {
    Int value;
}

Box^ box = new Box(value: 42);
Int n = box.value;
```

## 可选类型

`T?` 是 `Option<T>` 的短写，表示值可能为 `null`：

```c
Int? maybe = 42;
Option<Int> explicit = 42;
Int? none = null;
```

字段访问可以使用可选链：

```c
_ field = user?.name;
```

`??` 用于提供默认值：

```c
Int value = maybe ?? 0;
```

`.some(...)` 用于解包：

```c
if maybe is .some(value) {
    return value;
} else {
    return 0;
}
```

## 可错返回

`T@E` 是返回类型位置的 `Result<T, E>` 短写，表示函数成功时产生 `T`，失败时产生 `E`。`@E` 只出现在函数、方法和函数类型的返回位：

```c
enum Err {
    bad = 1,
}

Int@Err ok() {
    return 42;
}

Int main() {
    return try ok() catch () => 0;
}
```

可错值不会隐式变成 `T`。需要成功值时，要么在同错误类型的 `T@E` 函数中让错误传播，要么使用 `try expr catch (...) => 备用值` 显式处理错误分支。

## 数值转换

数值字面量在定型前可以由上下文决定类型。已经定型的数值之间不做隐式提升：

```c
Float f = 0.5;
Float a = 2 + f; // 2 是字面量，可按 Float 参与表达式

Int two = 2;
// Float b = two + f; // 编译错误：Int 变量不会隐式提升为 Float
Float b = Float(two) + f;
```

安全转换使用目标类型初始化形式，例如 `Float(two)`、`Int(f)`。普通代码不要用 `value$.as(Type)`
表示日常转换；`$.as` 是低层强制转换，主要用于裸指针、整数地址、外部函数接口或其他能力受控的底层场景。
