`trait` 描述一类类型必须提供的能力。它用于约束泛型和组织公共接口，不能直接写成普通变量、字段、参数或返回值类型。

## 定义 Trait

空 trait 可以写成声明：

```c
trait Numeric;
```

带方法要求的 trait 使用定义体：

```c
trait Equatable {
    static Bool equal(Self& lhs, Self& rhs);
}

trait Hashable: Equatable {
    UInt64 hash();
}
```

非 `static` 方法要求会使用当前类型作为接收者；`static` 方法要求没有 `self`。

## 动态 Trait 值（计划功能）

动态 trait 值是后续待完成的功能。计划中的设计会给每个 trait 提供三个编译器内建伴生类型：

```c
Writer.Any writer;
Writer.VTable vtable;
Writer.Receiver receiver;
```

`Any`、`VTable` 和 `Receiver` 是 trait 上的保留伴生类型名，不是用户定义的关联类型。
`Writer.Any` 是任意实现了 `Writer` 的值的借用动态视图。计划中的构造形式是 trait 上的
checked 操作：

```c
Writer.Any writer = Writer$.any(file);
```

这个构造要求 `typeof(file): Writer`；它不是 unchecked cast，也不会 move `file`。
`Writer.VTable` 是指向某个具体类型实现该 trait 时使用的方法表的轻量 handle：

```c
Writer.VTable file_vtable = Writer$.vtable(File);
```

`Writer.Receiver` 是实例方法 slot 使用的擦除 receiver：

```c
Writer.Receiver receiver = Writer$.receiver(file);
file_vtable.write(receiver, 123);
```

概念上，`Writer.Any` 存储一个 `Writer.Receiver` 和对应的 `Writer.VTable`。通过
`Writer.Any` 调用方法时会经由这张表动态分派：

```c
() write_one(Writer.Any writer) {
    writer.write(123);
}
```

传递 trait 方法指针时使用未绑定的 vtable slot 形式。receiver 方法会把擦除 receiver 作为
第一个参数；static 方法保留原来的参数列表：

```c
Fn<(), Writer.Receiver, Int> write_slot = writer.vtable.write;
write_slot(writer.receiver, 123);

Fn<Int> default_size = writer.vtable.default_buffer_size;
Int size = default_size();
```

## 在类型定义处实现

类型可以在声明头部列出满足的 trait，并在类型体内提供对应方法：

```c
trait HasValue {
    Int value();
}

struct Box: HasValue {
    Int inner;

    Int value() {
        return self.inner;
    }
}
```

仅仅“方法签名刚好匹配”不会自动满足 trait；类型需要显式声明它实现了哪个 trait。

## `extend`

`extend` 可以在类型定义之后补普通方法，或补显式 trait 实现：

```c
struct User {
    Int id;
}

extend User {
    Int id_value() {
        return self.id;
    }
}

extend User: HasValue {
    Int value() {
        return self.id;
    }
}
```

当前 `extend` 不支持 `init` 或 `deinit`。`extend Type: Trait { ... }` 表示给类型补一个 trait 实现，里面需要提供该 trait 要求的方法。

## 关联类型

Trait 可以声明关联类型：

```c
trait Iterable {
    associated Item;
    Item item();
}
```

关联类型可以带约束：

```c
trait HasItem {
    associated Item: Hashable & Equatable;
    Item item();
}
```

实现带关联类型的 trait 时，需要绑定关联类型：

```c
struct Counter: Iterable {
    associated Item = UInt8;

    UInt8 item() {
        return 42;
    }
}
```

`extend` 中也可以绑定：

```c
struct RangeCounter {}

extend RangeCounter: Iterable {
    associated Item = Int;

    Int item() {
        return 0;
    }
}
```

如果一个实现块涉及多个带同名关联类型的 trait，可以使用限定形式消歧：

```c
trait Left {
    associated Item;
    Item left();
}

trait Right {
    associated Item;
    Item right();
}

struct Pair: Left, Right {
    associated Left.Item = UInt8;
    associated Right.Item = Int;

    UInt8 left() {
        return 20;
    }

    Int right() {
        return 22;
    }
}
```

## Trait 约束

泛型约束使用 `@where(...)`：

```c
@where(T: Hashable & Equatable)
UInt hash_key<T>(T value);
```

关联类型绑定可以写在 trait 约束中：

```c
@where(T: Iterable<Item = UInt8>)
Int count<T>(T value);
```

也可以写相等约束：

```c
@where(T.[Iterable].Item == UInt8)
Int count_bytes<T>(T value);
```

## 内建 Trait

标准预导入模块提供一组常用 trait / 约束名，例如：

- `Numeric`：数值类型约束。
- `Mutable`：顶层 `!` 可变性约束。
- `Hashable`、`Equatable`：哈希和相等比较。
- `FromStringLiteral`：允许目标类型从字符串字面量初始化。
- `SubscriptGet` / `SubscriptSet`：让用户类型支持 `value[index]` 和 `value[index] = new_value`。

用户类型可以通过下面这组 trait 支持下标读写：

```c
public trait SubscriptGet {
    associated Index: Equatable;
    associated Value;

    Value subscript_get(Index index);
}

public trait SubscriptSet: SubscriptGet {
    () subscript_set(Index index, Value value);
}
```

读取 `value[index]` 需要 `SubscriptGet`。写入 `value[index] = new_value` 需要类型显式满足 `SubscriptSet`。

## 使用边界

- Trait 方法要求用来约束类型必须提供哪些方法，不能把 trait 当作普通对象类型使用。
- 子 trait 会继承父 trait 的方法要求和关联类型。
- 继承链不能出现未知父 trait、继承环或不兼容的同名关联类型。
- 关联类型约束建议写在 `@where(...)` 或 trait 约束中，保持泛型声明清晰。
