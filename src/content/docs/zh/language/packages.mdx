包是 Jiang 的项目边界。目录中放置 `package.ini` 后，编译器可以从包根模块
开始编译整个可达源码闭包。

## 包描述文件

使用固定文件名 `package.ini`：

```ini
[package]
name = app
version = 0.4.1
root = src/main.jiang
```

字段含义：

- `name`：包名称，未写时默认取目录名。
- `version`：包版本，可选。编译器接受 ASCII 字母、数字、`.`、`_`、`+` 和 `-`。
- `root`：入口源文件，未写时默认取 `<name>.jiang`。

当编译器输入路径是目录时，它读取该目录下的 `package.ini`，再编译 `root` 指向的源文件。

## 依赖

本地源码依赖写在 `[dependencies]` 中：

```ini
[package]
name = app
version = 0.4.1
root = src/main.jiang

[dependencies]
util = ../util
```

在这个包内，通过依赖别名导入：

```c
import util;

Int main() {
    return util.answer();
}
```

不带引号的 `import util;` 会优先匹配当前包的 `[dependencies]`。命中后，编译器读取
依赖包的描述文件，并把依赖包的根模块加入同一轮编译闭包。

## 公开接口

跨包访问只看依赖包根模块的公开命名空间：

- root 文件里的 `public` 声明是包的公开接口。
- root 文件里的 `public import` 可以重新导出一个模块命名空间。
- root 文件里的 `public alias` 可以重新导出一个具体 public symbol，例如
  `public alias answer = util.answer;`。
- 非 root 文件中的 `public` 声明不会自动成为包的公开接口。

因此包根模块是明确的公开接口边界。

## 导入边界

跨包不能用带引号路径直接导入依赖源码：

```c
import "../util/src/util.jiang"; // 错误：跨包文件导入
```

应该在 `package.ini` 中声明依赖，然后写：

```c
import util;
```

包依赖不能成环，但模块导入当前允许成环。

## 入口函数

只有根包的根模块会贡献可执行程序入口。依赖包即使定义了 `main`，
也只是普通函数。
