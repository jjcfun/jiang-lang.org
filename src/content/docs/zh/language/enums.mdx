Jiang 把有限标签集合和带携带值的联合类型分开表达：

- `enum` 表示有限命名成员集合。
- `union` 表示安全的带标签联合类型，每个变体可以携带值。

## 枚举

`enum` 成员按命名值使用：

```c
enum Mode {
    read,
    write,
}

Mode mode = Mode.read;
Mode other = .write; // 目标类型已知时可省略类型名
```

成员可以指定整数值：

```c
enum Priority {
    low = 1,
    medium,
    high,
}
```

`enum` 的 discriminant 默认使用 `Int32`。需要读取整数值时，使用目标整数类型构造，
例如 `Int(value)` 或 `UInt8(value)`；`enum` case 不再提供 `.value` 成员。

需要固定存储表示时，可以写 `enum(UInt8)` 或其他整数类型：

```c
enum(UInt8) ByteTag {
    block = 1,
    item,
}

UInt8 tag = UInt8(ByteTag.block);
```

`enum` 可以定义 `static` 方法和实例方法：

```c
enum Mode {
    read,
    write,

    static Int answer() {
        return 42;
    }

    Int value() {
        return Int(self);
    }
}

Int a = Mode.answer();
Int b = Int(Mode.read);
```

## 联合类型

`union` 是带标签的安全联合类型。它不是 C 风格的无标签裸联合。

```c
union Result<T, E> {
    T ok;
    E err;
}

Result<Int, ParseErr> a = Result.ok(42);
Result<Int, ParseErr> b = .err(ParseErr.bad);
```

携带值可以是普通类型，例如 `struct`、元组、数组、切片、`Fn<...>`、`T^`、`T&`、`T[*]`、`T*`、可选类型，以及其他 `union` / `enum`。

同类型多个变体可以合并声明：

```c
union Value {
    Int int_value, code;
    Double float_value;
    UInt8[] text;
}
```

`union` 可以省略显式标签枚举，直接使用变体名区分不同情况；也可以复用已有 `enum` 作为标签：

```c
enum Kind {
    int_value,
    text,
}

union(Kind) TaggedValue {
    Int int_value;
    UInt8[] text;
}
```

## `is` 匹配

`is` 用于模式匹配。匹配 `union` 变体时可以解构携带值：

```c
if value is .int_value(_ n) {
    print(n);
} else {
    print(0);
}
```

绑定可以显式写 `_`，也可以写 `_!` 创建可变绑定：

```c
if value is .int_value(_! n) {
    n = n + 1;
}
```

可选值使用专门的 `some` 模式：

```c
if maybe is some item {
    print(item);
} else {
    print(0);
}
```

## `switch` 匹配

`switch` 按模式分支匹配。处理 `enum`、`union` 和可选值时，建议覆盖所有情况；需要兜底时使用 `else`。

```c
Int code = switch mode {
    .read => 1,
    .write => 2,
}
```

处理 `union` 携带值：

```c
Int result = switch value {
    .int_value(_ n) => n,
    .code(_ code) => code,
    .float_value(_) => 0,
    .text(_ bytes) => bytes.length,
}
```

处理可选值：

```c
Int value = switch maybe {
    some item => item,
    null => 0,
}
```

## 模式规则

当前 `is` 和 `switch` 分支根只接受：

- 可选值模式：`some item`。
- 变体模式：`.name(...)` 或 `Type.name(...)`。
- 字面量模式：例如 `null`、整数、字符、布尔值。

绑定和通配符只作为可选值或变体携带值的子模式使用。元组模式当前不作为 `is` / `switch` 的分支根；需要解构元组时，使用独立解构语句。
