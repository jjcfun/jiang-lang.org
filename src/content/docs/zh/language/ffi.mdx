`extern` 声明用于调用外部符号，例如 C 库函数或系统接口。使用外部函数接口时，要自己确认参数类型、内存布局和资源释放方式与外部接口一致。

## 外部声明块

外部函数和全局变量可以写在 `extern { ... }` 代码块中：

```c
extern {
    public Int puts(UInt8[*:0] text);
    public Int open(UInt8[*:0] path, Int options);
    public Int errno;
}
```

也可以写单条声明：

```c
extern public Int puts(UInt8[*:0] text);
public extern Int errno;
```

顶层 `extern` 和 `public` 都是声明修饰符。`public` 表示这个外部符号声明也作为当前模块公开接口对外可见。

## C 字符串

C 风格字符串指针使用 `UInt8[*:0]` 这类带 sentinel 的多元素裸指针表示。字符串字面量在
`UInt8[*:0]` 上下文中可以生成以 `\0` 结尾的只读全局数据：

```c
extern public Int puts(UInt8[*:0] text);

puts("hello from Jiang");
```

借用的哨兵字符串视图使用 `UInt8[:0]&`。C 接口需要哨兵多元素指针时使用
`$.ptr()`；Jiang 代码需要把非字面量 sentinel 视图当作普通字节切片时，显式写
`view.slice()` 或 `view$.as(UInt8[]&)`：

```c
UInt8[:0]& view = "hello";
puts(view$.ptr());

UInt8[]& bytes = view.slice();
```

如果 C 接口需要指针加长度，直接建模成指针参数和长度参数：

```c
extern public Int write(Int fd, UInt8[*] buf, Int count);
```

## 单对象裸指针

`T*` 是单对象裸指针，主要用于外部函数边界、二进制接口和低层能力场景。它不表达所有权，不参与自动析构，
也不默认自动解引用。

```c
Int value = 42;
Int* raw = value$.ptr();
Int copied = raw$.get();
```

写入单对象裸指针指向的目标需要被指向值顶层可变：

```c
Int! value = 0;
Int!* raw = value$.ptr();
raw$.set(42);
```

## 多元素裸指针

`T[*]` 也是裸指针，用于可按下标访问的一段连续地址，常见于 C 接口的缓冲区参数：

```c
extern public Int read(Int fd, UInt8[*] buf, Int count);
```

`T*` 不像 C 指针那样用于自由位置移动。需要按位置访问连续内存时使用 `T[*]`。`T[*]` 没有长度；
如果 Jiang 代码需要保存长度视图，优先封装为 `T[]&` 或 `{ ptr, len }` 风格的 `struct`。

## 资源边界

外部接口返回的内存是否需要释放、使用哪个分配器释放、是否允许别名写入，都不由 `extern` 声明自动表达。把这类资源包装成 Jiang `struct` 时，建议在包装类型中明确：

- 哪个字段拥有资源。
- 哪个字段只是视图或引用。
- `deinit` 如何释放资源。
- 是否禁止隐式复制。

```c
struct CBuffer {
    UInt8[*] data;
    Int length;

    deinit() {
        self.data$.free();
        return ();
    }
}
```
