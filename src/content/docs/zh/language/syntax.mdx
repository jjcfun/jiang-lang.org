Jiang 的语法围绕“声明先行、类型显式、低层能力可见”来组织。读一个源文件时，通常先看导入和顶层声明，再进入函数体内的语句和表达式。

## 命名风格

项目采用下面的命名约定：

- 类型名使用 `PascalCase`。
- 函数名、变量名、字段名、枚举成员和模块别名使用 `snake_case`。
- `Int`、`Bool`、`UInt8` 等基本类型名按普通类型名使用。

```c
enum TokenKind {
    kw,
    string_lit,
    left_paren,
}

struct SourceFile {
    UInt8[]& file_path;
    Int start_offset;
}

UInt8[]& read_source(UInt8[]& file_path) {
    return file_path;
}
```

`Self` 是类型位置里的特殊名字。`self` 只在实例方法和 `init` 函数体内可用，表示当前接收者或正在初始化的目标；`static` 方法没有 `self`。

## 源文件结构

一个 Jiang 源文件由一组顶层声明组成。常见顶层声明包括：

- `import`：导入模块。
- `alias`：定义类型别名或已有符号别名。
- 全局变量声明。
- 函数声明或定义。
- `struct`、`enum`、`union`、`trait`。
- `extend`：给已有类型补方法或补 `trait` 实现。

```c
import math = "utils/math.jiang";

alias Byte = UInt8;
alias answer = math.answer;

public struct Pair {
    Int left;
    Int right;
}

public Int add(Int left, Int right) {
    return left + right;
}
```

`public` 标记声明对其他模块可见。没有 `public` 的声明默认只在当前模块内可见。

## 声明

函数声明总是先写返回类型，再写函数名、泛型参数、参数列表和函数体：

```c
Int add(Int left, Int right) {
    return left + right;
}
```

没有有意义返回值时，返回类型写成 `()`：

```c
() log_done() {
    print("done");
    return ();
}
```

## 语句

函数体和普通代码块中可以写局部变量声明、解构声明、赋值、调用语句、控制流语句和 `defer`。

```c
Int main() {
    Int! total = 0;

    for i in 0..10 {
        total = total + i;
    }

    return total;
}
```

局部声明以类型开头。使用 `_` 可以从初始化器推导类型：

```c
_ answer = 42;
_ name = "Jiang";
Int! count = 0;
```

语句通常以分号结束。`if`、`switch`、`while`、`for` 和代码块这类控制结构自身带 `{ ... }`，不在结构后额外加分号。

## 代码块和尾表达式

代码块使用 `{ ... }`。当代码块被当成表达式使用时，它的值来自最后一个不带分号的尾表达式；没有尾表达式时，代码块的值是单元值，也就是 `()`。

```c
Int value = {
    Int base = 40;
    base + 2
}
```

如果最后一行写成 `base + 2;`，它就是语句，不再贡献代码块的返回值。

## 表达式

Jiang 表达式包括字面量、名字、函数调用、字段访问、下标、切片、算术、比较、`if`、`switch`、`try catch` 和代码块。

```c
Int value = if flag {
    1
} else {
    2
}
```

字段字面量需要在有类型上下文时使用：

```c
struct Point {
    Int x;
    Int y;
}

Point origin = Point(x: 0, y: 0)
```

## 隐式操作层 `$`

`$` 用来访问值或类型的低层操作。常见用途包括显式解引用、取得引用或指针，以及查询类型大小。

```c
Int value = 42;
Int& ref = value$.ref();
Int copied = ref$.get();

UInt size = Int$.size();
```

`$` 绑定在左侧完整表达式上。如果要对复合表达式使用隐式层，使用括号：

```c
Int raw = (left + right)$.as(Int);
```

普通安全转换优先写成目标类型初始化，例如 `Float(value)`；不要把 `value$.as(Type)` 当作日常转换写法。
`$.as` 更适合裸指针、整数地址、FFI 等底层强制转换。

## 模式检查入口

`is` 用于模式匹配。可选值可以使用 `some` 解包：

```c
if maybe is some value {
    return value;
} else {
    return 0;
}
```

如果解包后的绑定本身需要可变，使用 `some _!`：

```c
if maybe is some _! value {
    value = value + 1;
}
```

`switch` 也使用模式。匹配 `enum`、`union` 和可选值时，应覆盖所有可能情况，或者使用 `else` 兜底。
