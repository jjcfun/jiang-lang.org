`trait` describes capabilities a type must provide. Use traits to constrain generics and organize shared APIs. A trait is not a normal value type.

## Defining Traits

Empty traits can be declared with a semicolon:

```c
trait Numeric;
```

Traits with required methods use a body:

```c
trait Equatable {
    static Bool equal(Self& lhs, Self& rhs);
}

trait Hashable: Equatable {
    UInt64 hash();
}
```

Non-`static` required methods use the current type as the receiver. `static` required methods do not have `self`.

## Dynamic Trait Values (Planned)

Dynamic trait values are planned but not implemented yet. The intended design gives every trait
three compiler-provided companion types:

```c
Writer.Any writer;
Writer.VTable vtable;
Writer.Receiver receiver;
```

`Any`, `VTable`, and `Receiver` are reserved companion type names on traits; they are not user-defined
associated types. `Writer.Any` is a borrowed dynamic view of any value whose concrete type implements
`Writer`. The planned construction is a checked trait operation on the trait:

```c
Writer.Any writer = Writer$.any(file);
```

The construction requires `typeof(file): Writer`; it is not an unchecked cast and it does not move
`file`. `Writer.VTable` is a lightweight handle to the method table for one concrete implementation:

```c
Writer.VTable file_vtable = Writer$.vtable(File);
```

`Writer.Receiver` is the erased receiver used by instance-method slots:

```c
Writer.Receiver receiver = Writer$.receiver(file);
file_vtable.write(receiver, 123);
```

`Writer.Any` conceptually stores a `Writer.Receiver` plus the matching `Writer.VTable`. Calls through
`Writer.Any` dispatch through that table:

```c
() write_one(Writer.Any writer) {
    writer.write(123);
}
```

Passing a trait method pointer uses the unbound vtable slot form. Receiver methods include the
erased receiver as their first parameter, while static methods keep their declared parameter list:

```c
Fn<(), Writer.Receiver, Int> write_slot = writer.vtable.write;
write_slot(writer.receiver, 123);

Fn<Int> default_size = writer.vtable.default_buffer_size;
Int size = default_size();
```

## Implementing in a Type Body

Types can list traits in their declaration and provide matching methods:

```c
trait HasValue {
    Int value();
}

struct Box: HasValue {
    Int inner;

    Int value() {
        return self.inner;
    }
}
```

## `extend`

`extend` adds methods or a trait implementation after the type declaration:

```c
struct User {
    Int id;
}

extend User {
    Int id_value() {
        return self.id;
    }
}

extend User: HasValue {
    Int value() {
        return self.id;
    }
}
```

`extend` members follow `member_decl`; `init` and `deinit` are not member declarations.

Extensions can be generic:

```c
@where(T == Int)
extend Holder<T> {
    public static Int only_int() {
        7
    }
}
```

Use `public extend` when an extension should be available to modules that import this module.
`public import` re-exports public extensions along with the imported module API.

Built-in type suffixes participate in the same extension lookup. For example, `UInt8[]^` looks for
static members on `Box<UInt8[]>`:

```c
@where(T == UInt8[])
extend Box<T> {
    public static Int byte_array_marker() {
        1
    }
}

Int marker = UInt8[]^.byte_array_marker();
```

## Associated Types

```c
trait Iterator {
    associated Element;
    Element? next();
}
```

Associated types can have bounds:

```c
trait HasItem {
    associated Element: Hashable & Equatable;
    Element item();
}
```

Implementations bind associated types:

```c
struct Counter: Iterator {
    associated Element = UInt8;

    UInt8? next() {
        return 42;
    }
}
```

`extend` can bind them too:

```c
struct RangeCounter {}

extend RangeCounter: Iterator {
    associated Element = Int;

    Int? next() {
        return 0;
    }
}
```

Use a qualified associated type path when multiple traits expose the same associated name:

```c
trait Left {
    associated Item;
    Item left();
}

trait Right {
    associated Item;
    Item right();
}

struct Pair: Left, Right {
    associated Left.Item = UInt8;
    associated Right.Item = Int;

    UInt8 left() {
        return 20;
    }

    Int right() {
        return 22;
    }
}
```

## Trait Bounds

```c
@where(T: Hashable & Equatable)
UInt hash_key<T>(T value);
```

Associated type binding:

```c
@where(T: Sequence<Element = UInt8>)
Int count<T>(T value);
```

Projected equality:

```c
@where(T.[Sequence].Element == UInt8)
Int count_bytes<T>(T value);
```

## Built-in Traits

The prelude provides common trait and bound names such as:

- `Numeric`
- `Mutable`
- `Hashable`
- `Equatable`
- `FromStringLiteral`
- `Iterator` / `Sequence`
- `SubscriptGet` / `SubscriptSet`

Subscript traits can model `value[index]`:

```c
public trait SubscriptGet {
    associated Index: Equatable;
    associated Value;

    Value subscript_get(Index index);
}

public trait SubscriptSet: SubscriptGet {
    () subscript_set(Index index, Value value);
}
```
