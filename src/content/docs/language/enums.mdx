Jiang separates finite tag sets from payload-bearing sum types:

- `enum` defines named members.
- `union` defines a safe tagged union where variants may carry payloads.

## Enums

```c
enum Mode {
    read,
    write,
}

Mode mode = Mode.read;
Mode other = .write;
```

Members may specify values:

```c
enum Priority {
    low = 1,
    medium,
    high,
}
```

Enum discriminants use `Int32` by default. Convert an enum value with a target integer
constructor such as `Int(value)` or `UInt8(value)`; enum cases do not expose a `.value`
member.

Use `enum(UInt8)` or another integer type when the stored representation must be
fixed:

```c
enum(UInt8) ByteTag {
    block = 1,
    item,
}

UInt8 tag = UInt8(ByteTag.block);
```

Enums can define `static` and instance methods:

```c
enum Mode {
    read,
    write,

    static Int answer() {
        return 42;
    }

    Int value() {
        return Int(self);
    }
}

Int a = Mode.answer();
Int b = Int(Mode.read);
```

## Unions

```c
union Result<T, E> {
    T ok;
    E err;
}

Result<Int, ParseErr> a = Result.ok(42);
Result<Int, ParseErr> b = .err(ParseErr.bad);
```

Same-type variants can be grouped:

```c
union Value {
    Int int_value, code;
    Double float_value;
    UInt8[]& text;
}
```

Unions can bind an explicit tag enum:

```c
enum Kind {
    int_value,
    text,
}

union(Kind) TaggedValue {
    Int int_value;
    UInt8[]& text;
}
```

## `is`

```c
if value is .int_value(_ n) {
    print(n);
} else {
    print(0);
}
```

Use `_!` for a mutable inferred binding:

```c
if value is .int_value(_! n) {
    n = n + 1;
}
```

Use `ref T name` to borrow a union payload instead of binding it by value:

```c
union Node {
    (Int, Int) pair;
    () empty;
}

Int read_ref(Int& value) {
    value$.get()
}

Int first(Node node) {
    if node is .pair(ref Int left, _) {
        return read_ref(left);
    }
    return 0;
}
```

This matters when the payload is large, non-copyable, or should be inspected without moving it.
`ref Int! value` is also allowed when the payload reference binding itself must be reassignable; the
`!` applies to the binding, not to the payload source.

Optional values use `.some(...)` and `.none`:

```c
if maybe is .some(payload) {
    print(payload);
} else {
    print(0);
}
```

## `switch`

```c
Int code = switch mode {
    .read => 1,
    .write => 2,
}
```

Union payloads:

```c
Int result = switch value {
    .int_value(_ n) => n,
    .code(_ code) => code,
    .float_value(_) => 0,
    .text(_ bytes) => bytes.length,
}
```

Borrowed union payloads use the same `ref` sub-pattern:

```c
Int result = switch node {
    .pair(ref Int left, _) => read_ref(left),
    .empty => 0,
}
```

Optional values:

```c
Int value = switch maybe {
    .some(payload) => payload,
    .none => 0,
}
```

## Pattern Roots

`is` and `switch` branch roots accept:

- optional patterns: `.some(payload)` and `.none`
- variant patterns: `.name(...)` or `Type.name(...)`
- literal patterns such as `null`, numbers, chars, and booleans

Bindings and wildcards are sub-patterns of optional or variant payloads. Tuple patterns are not switch roots; use destructuring statements for tuples.
