---
title: Enums & Pattern Matching
description: Enums with associated data, if-case, and switch pattern matching in Jiang.
---

Jiang `enum` supports associated data, carrying values of different types and arities. Enum variants are named subtypes, destructured on demand via pattern matching.

## Basic Definition

```c
enum Foo {
  A,
  B,
  C(Int, Int),
  D { x: Int, y: Int },
}
```

Variant associated values can be positional or named:

```c
// Create a variant with positional values
Foo my_variant = Foo::C(1, 2);

// Create a variant with named values
Foo my_other = Foo::D { x: 3, y: 4 };

// Named enum variant supports type inference
Foo my_record = Foo::D(y: 2, x: 0);
```

## if-case Matching

Match a single variant:

```c
if result is Foo::C(x, y) {
  print("x = %d y = %d", x, y);
}
```

Bind as mutable directly:

```c
if result is Foo::C(x, y!) {
  y += 100;
  print("x = %d y = %d", x, y);
}
```

## switch Matching

```c
_ tag = switch result {
  Foo::A => 0,
  Foo::B => 1,
  Foo::C(x, y) => x + y,
  Foo::D(x, y) => x * y,
};
```

switch can also use block form:

```c
switch result {
  Foo::A => {
    print("A");
  }
  Foo::B => {}
  Foo::C(x, y) => {
    print("%d + %d = %d", x, y, x + y);
  }
  Foo::D(y, x) => {}
}
```

### switch must be exhaustive

The compiler checks that switch covers all variants:

```c
// Compile error: missing variant Foo::D
_ tag = switch result {
  Foo::A => 0,
  Foo::B => 1,
  Foo::C(x, y) => x + y,
};
```

Use `default` or `_` as a catch-all:

```c
_ tag = switch result {
  Foo::A => 0,
  _ => 1,
};
```

## Enum variants with records

```c
enum Option {
  Some(Int, Int),
  None,
}

Option a = Option::Some(5, 2);

// Matching a variant with positional data
if a is Option::Some(x, y) {
  print("this is %d", x + y);
}
```

Accessing an enum variant directly infers the target enum type:

```c
// a has type Some, while b has type Option
_ a = Option::Some(5, 2);
Option b = Option::Some(5, 2);
