Generics let one declaration work with multiple types. Generic parameters are written after the
declaration name. Type parameters use a name such as `T`; const generic parameters use
`const Type Name`.

## Generic Functions

```c
T id<T>(T value) {
    return value;
}

Int a = id<Int>(42);
_ b = id(42);
```

## Generic Types

```c
struct Pair<A, B> {
    A first;
    B second;
}

Pair<Int, UInt8[]&> pair = Pair<Int, UInt8[]&> {
    first: 1,
    second: "one",
}
```

Type arguments may include suffixes:

```c
Pair<Int?, UInt8[]&> maybe_pair;
Pair<Int!, Int[3]^> mutable_and_owner;
```

## Const Generic Parameters

Const generic parameters are compile-time values:

```c
struct StaticCount<const Int N> {
    Int value;
}

Int const_value<const Int N>() {
    N
}
```

Const generic names are value-level names. They can be used in expressions, but not as type names.

## `@where`

`@where(...)` is a leading annotation:

```c
@where(T: Numeric)
T add<T>(T left, T right) {
    return left + right;
}
```

Multiple constraints use commas:

```c
@where(T: Hashable, U: Equatable)
Pair<T, U> make_pair<T, U>(T left, U right);
```

Intersection bounds use `&`:

```c
@where(T: Hashable & Equatable)
UInt hash_key<T>(T value);
```

## Equality Constraints

Generic parameter equality:

```c
@where(T == UInt8)
Int use_byte<T>(T value);
```

Use `!=` to reject a concrete type shape:

```c
@where(T != Box<_>)
struct PlainSlot<T> {
    T value;
}
```

Negative trait bounds use `!` before the trait name:

```c
@where(T: !Mutable)
struct ImmutableOnly<T> {
    T value;
}
```

Projected associated type equality uses `T.[Trait].Assoc == Type`:

```c
@where(T.[Sequence].Element == UInt8)
Int count_bytes<T>(T value);
```

## Associated Type Bindings

Trait bounds can bind associated types:

```c
@where(T: Sequence<Element = UInt8>)
Int count<T>(T value);
```

## Mutable Generic Parameters

Top-level mutability can be expressed through a `Mutable` bound:

```c
@where(T: Mutable)
struct Slot<T> {
    T value;
}

Slot<Int!> slot = Slot<Int!> { value: 1 }
```

## Errorable Returns

Generic functions can return `T@E` in result position:

```c
@where(T: Numeric)
T@ParseErr parse_number<T>(UInt8[]& text);
```

If the success value is optional, put `?` on the left side of `@`:

```c
T?@ParseErr parse_optional<T>(UInt8[]& text);
```

## Grammar Boundaries

- Generic type parameters accept types. Const generic parameters accept compile-time values with an
  explicit type, such as `const Int N`.
- `@where(...)` is written before the declaration, not after it.
- Associated type projection constraints use the `T.[Trait].Name == Type` form.
