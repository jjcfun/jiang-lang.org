Use `extern` declarations to call external symbols such as C library functions or system APIs. When using FFI, make sure your parameter types, memory layout, and resource cleanup match the external API.

## Extern Blocks

```c
extern {
    public Int puts(UInt8[*:0] text);
    public Int open(UInt8[*:0] path, Int options);
    public Int errno;
}
```

Single declarations can use top-level modifiers:

```c
extern public Int puts(UInt8[*:0] text);
public extern Int errno;
```

## C Strings

C-style string pointers use sentinel many pointers such as `UInt8[*:0]`. A string literal in that
context can be emitted as null-terminated read-only data:

```c
extern public Int puts(UInt8[*:0] text);

puts("hello from Jiang");
```

Borrowed sentinel string views use `UInt8[:0]&`. Use `$.ptr()` when a C API needs the sentinel
many pointer. Use `view.slice()` or `view$.as(UInt8[]&)` when Jiang code needs to treat a
non-literal sentinel view as a plain byte slice:

```c
UInt8[:0]& view = "hello";
puts(view$.ptr());

UInt8[]& bytes = view.slice();
```

When a C API needs a pointer plus a length, model the pointer and length explicitly:

```c
extern public Int write(Int fd, UInt8[*] buf, Int count);
```

## Raw Pointers

`T*` is a single-object raw pointer for FFI, ABI, and low-level code:

```c
Int value = 42;
Int* raw = value$.ptr();
Int copied = raw$.get();
```

Writing through a single-object raw pointer requires a mutable pointee:

```c
Int! value = 0;
Int!* raw = value$.ptr();
raw$.set(42);
```

## Many Raw Pointers

`T[*]` is also a raw pointer form. It is useful for C buffer parameters:

```c
extern public Int read(Int fd, UInt8[*] buf, Int count);
```

`T*` is not used for free position movement like a C pointer. Use `T[*]` for position-based access
into contiguous memory. `T[*]` has no length. Wrap it with a length field or use `T[]&` when Jiang
code needs a borrowed length-bearing view.

## Resource Boundaries

When wrapping external resources, make ownership explicit:

```c
struct CBuffer {
    UInt8[*] data;
    Int length;

    deinit() {
        self.data$.free();
        return ();
    }
}
```
